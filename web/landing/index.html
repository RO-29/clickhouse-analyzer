24 collectors · one poll loop

Built around how ClickHouse actually breaks

Collectors are grouped by failure domain, not by system table. Each maps to a real, diagnosable condition — with per-instance capability detection so a check is skipped, never guessed, when a table or column isn't available on that version or edition.

Parts & merges

The TooManyParts precursor chain — per-partition part ceilings, stalled merge pools, over-partitioning, and stuck mutations.

  • parts_to_throw
  • merges-stalled
  • mutations

Replication & Keeper

Replica queue depth and absolute delay, replication-queue exceptions, and Keeper session expiry — the moment replicas flip read-only.

  • absolute_delay
  • queue_size
  • is_expired

Inserts & backpressure

DelayedInserts throttling, new TOO_MANY_PARTS rejections (counted per interval, not lifetime), async-insert queue depth and flush failures, small-insert anti-patterns.

  • DelayedInserts
  • RejectedInserts Δ
  • async flush

Queries

Long-running and zombie HTTP queries, timeout and exception rates (rate-normalized, not raw counts), p95 latency vs a rolling baseline, and query storms.

  • p95 latency
  • error rate
  • zombie HTTP

Storage & system

Disk fill (object-storage disks excluded), S3 per-request latency, background pools, cache hit rates, system.errors, and crash/restart detection with pre-restart forensics.

  • disk fill
  • bg pools
  • crash_log

Schema & data flow

Per-instance schema drift, TTL health, dictionary load state, projection build gaps, materialized-view failures, and per-table insert freshness.

  • schema drift
  • MV failures
  • freshness
Alerts you can trust

Every alert is reproduced in a test before it can page you

Most monitoring tools ship alerts nobody has validated. This one runs a correctness harness: collectors are driven through known ClickHouse states by an in-process fake server, and a contract check fails the build if any query references a column, metric, or enum value that doesn't exist in the target version.

Fires on the real condition. RejectedInserts alerts on new rejections per interval — not a lifetime counter that stays critical forever after one event.
No dead queries. The SQL contract test catches wrong identifiers (like a pool metric missing a word) that would otherwise make a check silently never fire.
Version-aware. Capability detection gates each check per instance across OSS 23.3+ and ClickHouse Cloud 25.3+ — no false alarms from a missing table.
